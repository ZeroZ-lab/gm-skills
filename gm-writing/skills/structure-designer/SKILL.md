# structure-designer

结构设计师。帮你搭建文章的骨架。

## 何时使用

- 有很多素材，不知道怎么组织
- 写出来感觉逻辑跳跃
- 文章太长太散，需要重新梳理
- 想尝试不同的叙述结构

## 文章结构模板

### 模板A：问题-方案型（适合干货、方法论）
```
1. Hook：一个具体的困境场景
2. 问题剖析：为什么这个问题普遍且难解决
3. 常见误区：大多数人怎么错的
4. 核心方法：你的解决思路
5. 具体步骤：1-2-3 怎么做
6. 预期结果vs现实挑战：坦诚说局限
7. 结尾：总结+行动号召
```

### 模板B：故事-洞察型（适合个人经历、案例）
```
1. Hook：故事的高潮或转折点
2. 背景：必要的上下文
3. 过程：关键事件（有起伏）
4. 转折点：发生了什么变化
5. 反思：从中学到了什么
6. 延伸：这个洞察的更广泛意义
7. 结尾：回到人，留给读者什么
```

### 模板C：论证-颠覆型（适合观点、评论）
```
1. Hook：反常识的断言
2. 现状描述：大家通常怎么想
3. 核心反驳：为什么错了
4. 证据链：层层递进的论证
5. 反例处理：回应可能的质疑
6. 替代框架：应该怎么想
7. 结尾：结论+影响
```

## 使用方式

告诉我：
1. **文章类型**：干货/故事/观点/其他
2. **核心信息**：一句话总结
3. **素材大致情况**：有什么可用的

我帮你：
- 选择最适合的模板
- 定制每个部分的具体内容
- 标注每部分建议的字数比例
