# 需求文档：内容创作者技能套件生成器

## 问题

内容创作者（自媒体博主、公众号作者、小红书创作者等）有成熟的工作流，但：
1. 不知道如何把自己的经验转化为 AI 可用的 skills
2. 技术型 skill 编写指南（如 superpowers/writing-skills）门槛太高，需要理解 TDD、pressure testing 等概念
3. 业务型 skills 和开发型 skills 需求不同：不需要严谨测试，更注重创意输出和风格一致性

## 目标

创建一个**面向内容创作者的元技能**，让他们能用自然语言把自己的工作流转化为一套可复用的 skills。

## 核心工作流

目标用户的工作流：
```
选题 → 调研 → 写初稿 → 改稿 → 起标题 → 配图 → 发布
```

## 需求列表

### R1: 引导式 Skill 生成
- 通过对话引导创作者定义每个工作环节
- 不需要创作者理解 skill 技术细节
- 自动生成符合规范的 SKILL.md

### R2: 创作者友好的模板
- 提供内容创作专用的 skill 模板（hook、改写、配图提示等）
- 模板语言平实，没有技术术语
- 包含示例和填空式指导

### R3: 套件打包
- 把多个相关 skills 打包成一个「创作者套件」
- 提供套件索引和使用指南
- 支持版本迭代

### R4: 快速验证
- 提供简化的验证方式（不需要 pressure testing）
- 通过实际使用测试技能效果
- 支持快速迭代调整

## 成功标准

1. 一个内容创作者能在 30 分钟内，从零创建自己的第一个 skill
2. 生成的 skills 结构规范，能被 Claude Code 正确加载
3. 套件模式让多个 skills 协同工作

## 范围边界

**In Scope:**
- 元技能本身（gm-creator-kit）
- 内容创作专用模板库
- 套件生成和打包逻辑

**Out of Scope:**
- 通用的 skill 测试框架（复用现有）
- 技能市场/分享平台
- 非内容创作领域的 skill 生成

## 关键决策

1. **技术门槛**: 完全隐藏 skill 技术细节，用「工作流环节」代替「skill 结构」
2. **测试策略**: 业务型 skills 不需要 pressure testing，用「样例输出验证」代替
3. **输出格式**: 生成标准 skill 结构，确保兼容性
