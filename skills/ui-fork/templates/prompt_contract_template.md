# Prompt Contract Template

后续基于该设计系统继续生成页面时，必须遵守以下合同。

## 1. 必须继承

- 继承页面类型与信息架构，不允许直接换成另一种页面范式。
- 继承主阅读路径和主次区域关系。
- 继承已有组件 anatomy，不允许把组件改成自由拼贴。
- 继承 spacing、radius、border、typography、surface、state 和 motion 的 token 关系。
- 继承 light/dark 的主题变量关系；如存在多主题，不得为不同主题重做布局。

## 2. 可以变化

- 可以替换具体文案、图片、数据和业务内容。
- 可以按同一结构增加或减少同类模块。
- 可以在既有 token 范围内微调密度、强调层级和状态表达。
- 可以扩展新组件，但必须先复用已有组件结构和视觉规则。

## 3. 禁止破坏

- 不允许引入多个互相竞争的主色。
- 不允许擅自加入截图中没有证据的玻璃拟态、复杂发光、噪点纹理、大面积渐变或 3D 装饰。
- 不允许改变核心布局骨架来追求视觉新鲜感。
- 不允许让卡片、按钮、标签、列表等组件 anatomy 失控。
- 不允许把浏览、筛选、发现、操作等产品目标让位给纯视觉炫技。

## 4. 新页面生成规则

- 先继承页面类型与结构，再扩展局部视觉。
- 先复用已有模块骨架，再设计新模块。
- 先使用语义 token，再考虑新增 token。
- 新增状态必须覆盖 hover、focus、active、disabled、loading、empty、error 中与页面类型相关的状态。
- 所有变化必须建立在原系统连续性之上。
