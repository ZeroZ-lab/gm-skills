# 设计系统提炼结果

## 1. Design Brief

### 1.1 页面类型

说明页面属于 Marketing、Dashboard、Workspace、Directory、Content、Tool、Form、Mobile App 或混合类型。

### 1.2 页面目标

说明页面服务的产品目标或业务目标。

### 1.3 用户主任务

说明用户进入该页面最主要要完成什么。

### 1.4 主阅读路径

说明用户视线和操作路径如何展开。

### 1.5 页面气质关键词

只写能被结构、色彩、排版或组件规则支撑的关键词。

## 2. Structural System

### 2.1 页面一级结构

列出主要区域和顺序。

### 2.2 主次区关系

说明主内容区、辅助区、导航区、行动区之间的权重关系。

### 2.3 布局骨架

描述容器、列、卡片、侧栏、横向/纵向节奏。

### 2.4 栅格推断

给出栅格和断点推断；无法确认时标注为估计。

### 2.5 模块组织逻辑

说明模块如何分组、排序、重复和扩展。

### 2.6 信息密度策略

说明页面如何在信息量和可读性之间取舍。

## 3. Visual System

### 3.1 主题特征

说明整体主题策略，不写空泛赞美。

### 3.2 色彩策略

说明背景、表面、文字、强调色、状态色、边框的层级关系。

### 3.3 字体层级

说明标题、正文、辅助信息、标签、数字或数据文本的层级关系。

### 3.4 间距节奏

说明页面级、模块级、组件级间距规律。

### 3.5 圆角系统

说明不同组件的圆角层级和一致性规则。

### 3.6 边框与阴影系统

说明层级主要由边框、阴影、背景差还是留白建立。

### 3.7 Light/Dark 映射关系

如截图包含明暗主题，说明哪些是主题变量，哪些布局规则保持不变。

## 4. Component System

对每个重要组件输出以下字段：

- 作用：
- 结构：
- 层级：
- 复用规则：
- 可变项：
- 不可变项：

优先分析截图中真实存在的组件。常见组件包括：

- 导航栏
- Hero
- 搜索
- 按钮
- 标签/分类
- 卡片
- 列表/趋势/数据模块
- 侧栏
- 图表
- 分页
- 页脚

## 5. Interaction Assumptions

### 5.1 交互基调

说明交互应偏快速浏览、精确操作、沉浸阅读还是任务完成。

### 5.2 Hover / Focus / Active

说明可推断的状态变化规则。

### 5.3 状态切换

说明筛选、导航、标签、主题等切换规则。

### 5.4 动效建议

说明动效节奏、持续时间、使用边界。

### 5.5 主题切换规则

如适用，说明主题切换时布局、组件和 token 的继承关系。

## 7. Prompt Contract

### 7.1 后续 AI 设计必须遵守的规则

列出必须继承的页面类型、结构、组件、视觉和交互规则。

### 7.2 可以变化的部分

列出可替换内容、可扩展模块和可局部调整的视觉参数。

### 7.3 禁止破坏的骨架

列出不允许改变的布局范式、组件 anatomy、主题关系和信息架构。

## 8. Confidence Split

### 8.1 直接观察到的事实

只列截图中明确可见的内容。

### 8.2 高概率推断

列出基于截图和经验推断出的结论。

### 8.3 无法确认项

列出需要更多页面、交互状态或品牌资料才能确认的内容。
