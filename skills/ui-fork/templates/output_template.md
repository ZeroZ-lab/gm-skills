## A. Design Brief / 设计简报

### A1. 产品类型

说明页面属于 Marketing、Dashboard、Workspace、Directory、Content、Tool、Form、Mobile App、AI Workbench 或混合类型。必须区分主类型和次类型。

### A2. 页面目标与产品任务

说明页面服务的产品目标或业务目标，不只描述画面内容。

### A3. 用户主任务

说明用户进入该页面最主要要完成什么。

### A4. 主阅读路径与辅助路径

说明用户视线和操作路径如何展开。

### A5. 页面气质关键词

只写能被结构、色彩、排版或组件规则支撑的关键词。

### A6. 适用范围与不适用范围

说明该设计系统适合继续生成哪些页面，不适合套用到哪些页面。

## B. Design Guide / 设计指南

### B1. 设计总原则

提炼该界面背后的设计原则。每条原则必须解释它如何影响布局、组件、视觉或交互。

建议覆盖：

- 信息优先还是品牌优先
- 模块化程度
- 主题切换关系
- 信息密度策略
- 首页或当前页在产品中的角色
- 后续 AI 生成必须受哪些结构约束

### B2. 信息架构

必须说明：

- 页面一级结构
- 主次区关系
- 阅读路径
- 信息分组逻辑

### B3. 布局系统

必须说明：

- 总体布局模式
- 页面主容器、模块容器、内容单元容器
- 栅格、列宽、断点和响应式推断
- 页面级、模块级、组件级间距节奏
- 新增模块、删减模块、扩展页面时的结构连续性

### B4. 视觉语言

必须说明：

- 基础层 token 与主题层 token
- 背景、表面、文字、强调色、状态色、边框的层级关系
- 明暗主题关系，如适用
- 层级由背景对比、边框、阴影、留白、字体还是色彩建立
- 圆角、边框与阴影系统
- 会破坏原系统的视觉动作

### B5. 字体与排版规范

必须说明：

- 排版风格判断：平台型、营销型、数据型、编辑型、工具型等
- H1、H2、H3、Body、Caption、Meta、数字文本或代码文本的层级关系
- 标题长度、描述行数、截断规则、数字突出方式、meta 弱化方式和可读性约束

### B6. 组件系统

对每个重要组件使用同一结构输出：

| 字段 | 说明 |
| --- | --- |
| 作用 | 该组件解决什么产品或信息问题 |
| 结构 | 组件由哪些子元素组成 |
| 层级 | 在页面中的视觉和任务权重 |
| 复用规则 | 什么时候复用，如何复用 |
| 可变项 | 文案、数据、数量、状态等可变化部分 |
| 不可变项 | 不能破坏的 anatomy、尺寸、层级或关系 |
| 状态与交互 | hover、focus、active、selected、disabled 等 |
| 相关 tokens | color、spacing、radius、border、shadow、motion 等 |

优先分析截图中真实存在的组件。不要为了填模板虚构组件。

### B7. 交互规范

必须说明：

- 交互基调
- Hover / Focus / Active 规则
- 点击反馈与状态切换
- 动效节奏、持续时间、使用边界
- 主题切换时布局、组件和 token 的继承关系，如适用

### B8. 前端落地建议

当用户要求 `system` 模式或前端落地时输出。建议拆成：

- Design Tokens
- Semantic Tokens
- Component Spec
- Page Template
- Theme Strategy
- Responsive Rules

不要直接写完整前端代码，除非用户明确要求实现。

## C. Design Tokens Draft / 设计变量草案

说明：tokens 是草案，不是最终权威值。无法确认的值留空或标注“未知”；估计值必须标注“估计”。

必须覆盖：

- Color Tokens
- Typography Tokens
- Spacing Tokens
- Radius Tokens
- Border Tokens
- Shadow Tokens
- Motion Tokens
- Semantic Tokens
- Component Tokens

使用 `templates/tokens_template.yaml` 的结构。复杂 token 建议放入 fenced `yaml` code block。

## D. AI Prompt Contract / AI 生成约束

### D1. 后续 AI 设计必须遵守的规则

列出必须继承的页面类型、结构、组件、视觉和交互规则。

### D2. 可以变化的部分

列出可替换内容、可扩展模块和可局部调整的视觉参数。

### D3. 禁止破坏的骨架

列出不允许改变的布局范式、组件 anatomy、主题关系和信息架构。

### D4. 新增组件扩展规则

说明新增组件必须如何继承 spacing、radius、border、typography、state 和 motion 规则。

### D5. 禁止项

列出后续 AI 不得擅自引入的视觉语言、布局范式和交互模式。

## E. Confidence Split / 观察、推断与未知

### E1. 直接观察到的事实

只列截图中明确可见的内容。

### E2. 高概率推断

列出基于截图和经验推断出的结论。

### E3. 无法确认项

列出需要更多页面、交互状态或品牌资料才能确认的内容。
