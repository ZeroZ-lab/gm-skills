# AI 项目精选导航网站设计指南

**版本**：v1.0
**适用范围**：AI 工具导航站、项目发现平台、资源目录站、榜单/专题型内容平台
**样本来源**：白色版 + 黑色版首页视觉稿
**文档目标**：为后续 AI 生成页面、设计扩展、组件复用、前端落地提供统一约束

## 1. Design Brief / 设计定位

### 1.1 产品类型

这是一个典型的 **内容型 + 导航型 + 轻数据化展示** 的 AI 项目平台首页，不是强交互后台，也不是营销官网。

核心任务：

- 快速建立认知：告诉用户这是一个 AI 项目发现与筛选平台。
- 快速帮助浏览：分类、标签、趋势、专题让用户进入内容。
- 提高信息密度但不压迫：项目卡片很多，但整体仍然清晰。
- 支持持续扩展：后续可增加榜单、收藏、筛选、详情页、提交页等。

### 1.2 页面气质关键词

- 清晰：模块边界明确，主次路径稳定。
- 克制：强调色数量少，装饰不抢信息。
- 信息优先：卡片、标签、趋势和专题服务浏览发现。
- 平台感：首页像产品能力缩影，而不是单一宣传页。
- 高可扫描性：标题、标签、卡片 anatomy 和右侧栏共同降低浏览成本。

白色版偏内容平台、编辑化、易读和大众化。黑色版偏科技平台、AI 工具站、产品感和视觉张力。两者应视为同一设计系统的主题变量，不是两套布局。

## 2. Design Principles / 设计总原则

### 2.1 信息优先于装饰

页面中的颜色、阴影、边框、圆角都服务于信息组织。不要为了“AI 感”额外加入无依据的发光、噪点或玻璃拟态。

### 2.2 模块优先于自由排版

Hero、统计、分类、精选卡片、趋势侧栏、标签区、专题区都是标准容器模块，不是随意拼贴。后续扩展应继续使用模块化容器。

### 2.3 统一骨架，主题切换

白色与黑色主题共享信息架构、布局逻辑和组件结构，只切换主题 token。黑白主题是视觉皮肤，不是重新设计页面。

### 2.4 内容密度高，但认知负担低

用大标题定调、模块边界分组、统一卡片结构压缩信息，用右侧栏承担辅助发现路径，避免主内容被打断。

### 2.5 首页即平台缩影

首页同时表达项目发现、分类浏览、热门趋势、标签聚合、专题组织和数据背书，不能简化成单纯 hero + CTA。

## 3. Information Architecture / 信息架构

### 3.1 页面一级结构

1. 顶部导航栏
2. Hero 区
3. 数据统计区
4. 分类筛选区
5. 精选项目区
6. 全部项目区
7. 右侧趋势/标签/专题辅助区
8. 分页区
9. 页脚

### 3.2 页面阅读路径

导航 -> 标题价值主张 -> 主行动按钮 -> 分类入口 -> 精选项目 -> 全部项目 -> 右侧趋势补充。

左侧主列承担主浏览路径，右侧侧栏承担辅助发现路径，顶部搜索承担目标明确时的直达路径。

## 4. Layout System / 布局系统

### 4.1 总体布局模式

采用双列内容布局：左侧为主内容区，右侧为辅助信息区。这是平台首页结构，适合内容持续增长。

### 4.2 容器策略

- 页面主容器：约束整体最大宽度和左右留白。
- 模块容器：承载 hero、统计、分类、项目列表、趋势等大区块。
- 内容单元容器：项目卡、标签、趋势项、专题项。

### 4.3 栅格建议

- 桌面端适合 12 栏网格。
- 左主区约占 8-9 栏。
- 右侧栏约占 3-4 栏。
- 精选区适合 3 列大卡。
- 全部项目适合 5-6 列小卡。
- 右侧专题适合单列列表。

以上是基于截图的高概率推断，真实断点需要前端验证。

### 4.4 间距系统

建议使用 8pt 系统：4、8、12、16、20、24、32、40、48。

- 标签内边距：8/12。
- 卡片内部小间距：12/16。
- 模块内部标准间距：20/24。
- 大模块上下间距：32/40。
- 首屏大区间距：48+。

## 5. Visual System / 视觉语言

### 5.1 主题结构

基础层 token 包括字体、间距、圆角、阴影、边框粗细、组件尺寸。主题层 token 包括背景色、文本色、分割线色、主品牌色、hover 态和卡片表面色。

### 5.2 色彩体系

品牌主色是高亮蓝，用于 Logo、主按钮、当前导航、交互强调和趋势局部强调。

白色主题建议：

- Background：`#FFFFFF` 附近。
- Surface：`#FFFFFF` / `#FAFAFA` 附近。
- Border：`#E5E7EB` 附近。
- Primary Text：`#111827` 附近。
- Secondary Text：`#6B7280` 附近。
- Accent Blue：`#2563EB` 附近。

黑色主题建议：

- Background：`#020617` 到 `#07111F` 区间。
- Surface：`#0B1220` 到 `#0F172A` 区间。
- Border：`#1F2937` 或更低对比深边框。
- Primary Text：`#F8FAFC`。
- Secondary Text：`#CBD5E1`。
- Accent Blue：`#2563EB` / `#3B82F6` 附近。

这些值是截图估计值，不应视为最终品牌色。

### 5.3 层级建立方式

白色版不是无边界，黑色版也不是靠发光。两者都依赖背景与卡片表面对比、边框与分割线、文字层级与密度差异来建立秩序。

## 6. Typography / 字体与排版规范

该页面属于平台型 UI 排版，不是海报型排版。排版目标是快速扫描、标题清晰、数据醒目、描述不抢戏，并在大量卡片下保持统一。

建议层级：

- H1：首屏价值主张，粗体、大尺寸、紧凑行高。
- H2：模块标题，如“精选项目”“全部项目”“本周趋势”。
- H3：卡片标题，如项目名、专题名。
- Body：项目描述、模块说明、次级说明。
- Caption / Meta：标签、统计说明、页脚、计数信息。

规则：

- 标题一律短句，不写长段。
- 描述最多 2-3 行。
- 卡片标题优先单行或双行截断。
- Meta 信息统一弱化。
- 数字信息要比说明文字更突出。

## 7. Component System / 组件系统

### 7.1 顶部导航栏

- 作用：提供平台身份、一级导航、搜索直达和账号入口。
- 结构：Logo、一级导航、搜索框、登录/注册操作。
- 复用规则：导航项控制在 4-6 项，当前项使用颜色或下边线标识。
- 不可变项：搜索应保持高频可见，登录与注册保持主次按钮层级。

### 7.2 Hero 区

- 作用：功能型价值说明 + 行动引导。
- 结构：大标题、简述、主按钮、次按钮、右侧数据统计面板。
- 复用规则：大标题一句话说明平台价值，描述 1-2 行，CTA 不超过两个。
- 不可变项：不能变成纯品牌海报区，必须保留进入浏览或提交的行动路径。

### 7.3 数据统计卡

- 作用：构建平台已成型的信任感。
- 结构：图标、数值、指标名、指标说明。
- 复用规则：4 个左右最佳，数字层级高于描述，图标只做识别。

### 7.4 分类标签栏

- 作用：支持快速筛选和内容范围感知。
- 结构：一组 pill 标签按钮，当前筛选态高亮。
- 复用规则：标签短词化，激活态用品牌色，默认态用弱边框或弱底色。

### 7.5 项目卡片

项目卡片是主系统组件。

大卡片结构：

- Logo / 图标
- 项目名
- 精选标识
- 简述
- 标签
- 数据指标
- 外链 / GitHub 等动作

小卡片更紧凑，但必须保留相同骨架。

核心规则：

- 所有项目卡必须统一 anatomy。
- Logo 尺寸固定。
- 标题区域高度尽量统一。
- 描述行数限制一致。
- 标签数量限制。
- 底部 meta 信息结构统一。

### 7.6 趋势列表

- 作用：提供轻数据化辅助发现。
- 结构：排名序号、趋势名称、小型趋势线、增长百分比。
- 复用规则：只展示趋势信号，不展示完整统计细节；视觉要轻，不压过主内容。

### 7.7 标签云 / 热门标签

- 作用：压缩内容分类并提供横向发现入口。
- 结构：标签名、数量。
- 复用规则：标签尺寸统一，允许热度差异但不建议大小失控。

### 7.8 专题列表

- 作用：提供编辑化内容入口。
- 结构：缩略图、标题、简介、项目数量。
- 复用规则：专题图尺寸统一，信息密度低于项目卡。

### 7.9 分页组件

- 作用：支撑项目列表扩展。
- 结构：上一页、下一页、页码、当前态。
- 复用规则：当前页使用品牌色，位置在主内容底部中心或左对齐区块末端。

## 8. Interaction Guidelines / 交互规范

交互基调应轻、快、明确，不做过度动效。

- 按钮 hover：颜色加深或阴影微增强。
- 卡片 hover：边框增强或表面轻微抬升。
- 标签 hover：底色变化。
- 列表项 hover：背景微浮起。
- 链接 hover：颜色强化。
- 搜索 focus：边框或 focus ring 必须清楚。
- 动效：150-250ms，ease-out，不使用夸张位移或大面积 blur。

## 10. Frontend Handoff Notes / 前端落地建议

建议拆成四层：

- Design Tokens：color、spacing、radius、border、shadow、typography、z-index、motion。
- Semantic Tokens：`bg.default`、`bg.surface`、`text.primary`、`text.secondary`、`border.default`、`action.primary`。
- Component Spec：为每个组件定义 anatomy、states、size、spacing、variant、usage rules。
- Page Template：导航型首页模板、列表页模板、详情页模板、专题页模板。

## 11. AI Prompt Contract / 给 AI 的生成约束

后续 AI 继续基于该规范做页面时，必须遵守：

1. 保持平台型信息架构，不改为营销官网。
2. 保持双列结构，左主右辅。
3. 保持模块卡片化组织。
4. 保持高信息密度但统一骨架。
5. 保持蓝色为唯一主强调色。
6. 保持白/黑主题只是变量切换。
7. 项目卡片必须统一结构。
8. 所有组件圆角、边框、间距应属于同一系统。
9. 不加入多余渐变、发光、噪点、拟物效果。
10. 页面应优先服务浏览与发现，而不是品牌情绪表达。

## 12. Extraction Checklist / 关键提炼项

- 产品层：页面类型、产品目标、用户主任务、主要阅读路径。
- 结构层：信息架构、布局骨架、模块拆分、主次区关系。
- 视觉层：色彩体系、明暗主题、字体层级、边框/圆角/阴影策略。
- 组件层：导航、搜索、按钮、标签、项目卡、趋势列表、标签云、专题卡、分页。
- 交互层：hover、active、focus、主题切换、筛选切换。
- 实现层：tokens、semantic tokens、component anatomy、页面模板。

## 13. Core Design Conclusion / 核心设计结论

这组截图体现的不是单一风格，而是一种平台型界面设计系统原型：用模块化结构承载高密度信息，用统一卡片骨架保证扩展性，用极少数强调色维持秩序，用黑白双主题证明设计可变量化，并用首页完成品牌、导航、发现、筛选、数据背书的多任务整合。

## 14. Confidence Split / 观察、推断与未知

### 14.1 直接观察到的事实

- 存在白色版和黑色版首页视觉稿。
- 页面有顶部导航、Hero、项目卡片、分类标签、右侧辅助模块、分页和页脚。
- 蓝色承担主要强调色。
- 项目卡和标签形态重复出现。

### 14.2 高概率推断

- 这是 Directory / 导航聚合型页面，次类型是 Content / 轻数据平台。
- 页面适合以 Design Token + Theme Token 实现。
- 双列布局适合支撑后续趋势、专题、标签和推荐模块扩展。

### 14.3 无法确认项

- 真实字体、精确 hex、断点、hover/focus 状态无法仅凭静态图确认。
- 卡片点击区域、搜索联想、筛选联动、主题切换动画需要交互稿或前端实现确认。
