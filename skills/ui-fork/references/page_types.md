# 页面类型分支

先判断页面类型，再决定提炼重点。页面可能是混合类型，但必须说明主类型和次类型。

## Marketing / 官网型

用于品牌叙事、产品介绍、转化和信任建立。

重点提炼：

- 首屏叙事结构
- CTA 层级
- section 节奏
- 品牌色和情绪表达
- 社会证明、案例、功能展示模块
- 移动端转化路径

避免：

- 把营销页分析成后台工作台。
- 只提炼视觉，不提炼转化路径。

## Dashboard / 数据看板型

用于监控指标、发现异常、比较趋势和辅助决策。

重点提炼：

- 指标层级
- 图表和表格权重
- 筛选器位置和联动关系
- 时间范围、状态、趋势表达
- 空状态、异常状态和加载状态
- 高信息密度下的可读性

避免：

- 把图表当作普通卡片处理。
- 忽略数据状态和筛选逻辑。

## Workspace / 工作台型

用于完成多步骤任务、管理对象、协作和操作。

重点提炼：

- 导航和工作区关系
- 对象列表、详情、编辑区的组织方式
- 工具栏和快捷操作
- 状态反馈和权限提示
- 操作密度和快捷路径
- 多面板或多列布局

避免：

- 只关注外观，不分析任务流。
- 忽略操作状态和错误反馈。

## Directory / 导航聚合型

用于浏览、筛选、发现和跳转到资源。

重点提炼：

- 分类体系
- 搜索和筛选路径
- 卡片 anatomy
- 标签系统
- 主列表和辅助推荐区关系
- 分页、排序、趋势模块

避免：

- 改成营销落地页结构。
- 让不同卡片结构失控。

## Content / 内容阅读型

用于文章、文档、知识库、媒体内容消费。

重点提炼：

- 阅读宽度
- 标题、正文、目录、注释层级
- 内容导航和相关推荐
- 图文关系
- 长内容节奏
- 可访问性和可读性

避免：

- 信息密度过高破坏阅读。
- 把内容页做成卡片墙。

## Tool / 工具型

用于输入、生成、转换、计算、编辑或执行具体任务。

重点提炼：

- 输入区、输出区和控制区关系
- 默认值和参数组织
- 执行反馈
- 结果展示和导出路径
- 错误、空状态、处理中状态
- 快捷操作和重复使用路径

避免：

- 过度装饰影响任务效率。
- 忽略错误和结果状态。

## Form / 流程录入型

用于注册、配置、申请、结账、向导流程。

重点提炼：

- 步骤结构
- 字段分组
- 校验和错误提示
- 进度反馈
- 主按钮和返回路径
- 移动端输入体验

避免：

- 只分析静态布局，不分析校验和流程。
- CTA 层级不清。

## Mobile App / 移动端界面

用于小屏交互、单手操作和移动场景。

重点提炼：

- 顶部栏、底部导航和手势区域
- 屏幕层级
- 触控目标尺寸
- 列表和卡片密度
- 状态栏、安全区、底部吸附操作
- 页面转场和返回路径

避免：

- 直接套用桌面端栅格。
- 忽略触控、底部导航和安全区。
