# gm-writing-kit

完整写作流程套件。从灵光一闪到点击发布。

## 写作流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ topic-picker │ → │research-assistant│ → │ hook-crafter │
│   选题决策    │    │   调研整理    │    │   开头钩子    │
└─────────────┘    └─────────────┘    └─────────────┘
                                              ↓
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ publish-prep │ ← │ revision-tuner │ ← │ draft-generator │
│   发布准备    │ ← │   改稿调优    │ ← │   初稿生成    │
└─────────────┘    └─────────────┘    └─────────────┘
        ↑
┌─────────────┐
│structure-designer
│   结构设计   │
└─────────────┘
```

## 按场景选择

| 你现在处于 | 使用 |
|-----------|-----|
| 有个模糊想法，不知道写不写 | **topic-picker** |
| 确定了选题，缺素材 | **research-assistant** |
| 不知道怎么开头 | **hook-crafter** |
| 素材很多，不知道怎么组织 | **structure-designer** |
| 结构有了，要写成文 | **draft-generator** |
| 初稿完成，要修改 | **revision-tuner** |
| 定稿了，起标题准备发 | **publish-prep** |

## 完整流程示例

**Step 1: 选题**
> "我想写关于远程办公的，但角度太多不知道选哪个"
> → 使用 **topic-picker**

**Step 2: 调研**
> "选了'大厂为什么反对远程办公'这个角度，需要数据支撑"
> → 使用 **research-assistant**

**Step 3: 结构设计**
> "有素材了，怎么组织文章结构"
> → 使用 **structure-designer**

**Step 4: 写 Hook**
> "结构定了，需要一个抓人的开头"
> → 使用 **hook-crafter**

**Step 5: 生成初稿**
> "按结构填充内容"
> → 使用 **draft-generator**

**Step 6: 改稿**
> "初稿写完了，但感觉太平"
> → 使用 **revision-tuner**

**Step 7: 发布准备**
> "要发了，起标题+配图"
> → 使用 **publish-prep**

## 提示

- 不是每篇文章都需要走完整流程
- 小文章可以直接从 **draft-generator** 开始
- 如果卡在某一环节，可以跳回上一环节重新来
- 这套 skills 可以和你已有的 **gm-writing**、**gm-x-hook-writer** 配合使用
