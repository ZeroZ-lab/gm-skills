# gm-learn-80

帮助用户在 AI 时代快速掌握一个陌生领域的 80% 核心知识。

## Overview

把陌生领域快速压缩成**可进入、可理解、可应用**的学习结构。

**核心原则**：
- 不是做百科式讲解，而是先压缩成认知骨架
- 目标不是穷尽知识，而是获得理解力、判断力和应用入口
- 问题驱动，而非章节驱动
- 输出导向：每次都要给"如何验证是否学会"的方式

---

## When to Use

**适用场景**：
- "如何快速学习 XX？"
- "我想快速掌握这个领域"
- "帮我梳理这个领域的核心知识"
- "我只有几天时间，怎么学这个领域最有效？"
- "帮我做一个这个领域的认知地图"

**不要用于**：
- 需要博士学位级深度的学术准备
- 已经有完整课程体系的学习计划
- 只需要查一个具体概念的定义

---

## Core Pattern

### 输出结构（7个模块）

| 模块 | 核心问题 | 产出要求 |
|------|----------|----------|
| 1. 领域本质 | 这个领域在解决什么问题？核心价值？边界？ | 1-2 段话，清晰界定 |
| 2. 核心知识骨架 | 最重要的 5-10 个概念及关系 | 概念 + 层级关系图 |
| 3. 典型流程 | 做事的一般流程是什么？ | 输入→步骤→输出 |
| 4. 关键变量与误区 | 决定结果的关键杠杆？新手常见错误？ | 杠杆点 + 避雷指南 |
| 5. 高价值问题清单 | 最值得优先搞懂的 10-20 个问题 | 按入门→进阶排序 |
| 6. 最小实践项目 | 1-3 个 1-3 小时可完成的项目 | 目标+知识点+完成标准 |
| 7. 学习路径 | 7 天快速路径或分阶段路线 | 每天目标+练习+输出 |

### 质量标准

合格输出必须让用户：
1. 知道"这个领域到底在干什么"
2. 能说出最重要的几个概念及关系
3. 知道先学什么、后学什么
4. 知道学完可以做什么最小实践

**失败信号**：只是资料清单、术语堆砌或大而全综述

---

## Implementation

### Step 1：定义领域
先明确：
- 这个领域解决什么问题
- 它不是什么
- 与邻近领域如何区分

### Step 2：压缩骨架
提炼：
- 关键概念（最多 10 个）
- 关键结构
- 关键流程
- 关键变量

### Step 3：提取高频问题
输出：
- 用户最应该优先搞懂的问题列表
- 从"看懂"到"会用"的问题顺序

### Step 4：设计练习
输出：
- 最小练习（30 分钟内）
- 最小项目（1-3 小时）
- 自测题 / 复盘问题

### Step 5：给出学习路径
输出：
- 7 天计划
- 或分阶段路线图（入门/进阶/应用）

---

## Common Mistakes

| 错误 | 修正 |
|------|------|
| 不分层级地堆百科知识 | 先给骨架，再给血肉 |
| 输出过多无主次的资料 | 每个资料都要说明"为什么是现在看" |
| 用"定义+分类+历史"凑篇幅 | 问题驱动，而非章节驱动 |
| 给出空泛鼓励而没有结构 | "多看多练"→替换为具体动作 |
| 假装用户需要完整学术体系 | 优先高杠杆，再全覆盖 |

---

## Red Flags - STOP and Refactor

如果输出出现以下情况，立即停止并重构：

- ❌ 开头没有一句话核心结论
- ❌ 概念罗列超过 15 个且没有层级
- ❌ 没有具体的"完成标准"或"验证方式"
- ❌ 给出资料链接但没有筛选标准
- ❌ 缺少时间维度或阶段性目标
- ❌ 使用"还有很多内容需要学习"等开放式结尾

**All of these mean: 回到 Step 1，重新压缩。**

## Output Template

```markdown
# 结论
一句话说明：要快速掌握这个领域的 80%，最该抓住什么。

# 1. 这个领域的本质
- 核心问题
- 核心目标
- 与相邻领域的区别

# 2. 这个领域最重要的知识骨架
- 概念 1（上位/核心）
- 概念 2（方法/工具）
- 概念 3（结果指标）
- 概念关系说明

# 3. 典型流程
- 输入
- 处理步骤（最多 5 步）
- 输出
- 常见错误

# 4. 关键变量与常见误区
- 关键变量（影响结果的杠杆）
- 新手误区
- 可以后学的内容

# 5. 最值得优先搞懂的 10 个问题
按顺序列出，并说明重要性。

# 6. 3 个最小实践项目
每个项目：
- 项目目标
- 所需时间
- 涉及知识点
- 完成标准

# 7. 一个 7 天学习路径
按天列出：
- 学习目标
- 重点内容
- 一个练习
- 一个输出任务

# 8. 最后提醒
指出：这个领域里哪些内容是"看起来很重要但不必一开始就深挖"的。
```
